# צרו Agent ו-Skill משלכם

הסדנה נגמרה, אבל הכלים שלכם רק מתחילים. הנה פרומפטים מוכנים שאפשר להדביק ישירות ל-Claude Code כדי ליצור agent ו-skill בעצמכם — בלי לכתוב שום קוד.

---

## חלק 1: יצירת Agent חדש

### מה זה Agent?

Agent הוא עוזר אוטונומי שפועל בהקשר מבודד. Claude מחליט מתי להפעיל אותו, על בסיס ה-description שכתבתם. Agent מתאים במיוחד ל:

- ניתוח פתוח — מחקר, סריקה, הפקת תובנות
- משימות שדורשות כמה שלבים ברצף
- עבודה אוטונומית שאתם לא רוצים לנהל ידנית

### הפרומפט — הדביקו אותו ישירות ל-Claude Code:

```
אני רוצה ליצור agent חדש ב-Claude Code.

שם: [תאר שם קצר באנגלית, לדוגמה: sprint-summarizer]
מטרה: [תאר מה ה-agent אמור לעשות]
מתי להשתמש בו: [תאר באיזה מצב Claude צריך לפעיל אותו]
קלט: [מה ה-agent מקבל? קובץ? טקסט? מה?]
פלט: [מה אתה רוצה שיוציא?]

תיצור עבורי קובץ agent תקני עם:
- YAML frontmatter (name, description, model: inherit, color)
- הוראות ברורות לסוכן
- תבנית פלט בעברית
- עקרונות מנחים

שמור את הקובץ ב-.claude/agents/[שם].md
```

### רעיונות ל-Agents למנהלי מוצר

| Agent | מה הוא עושה |
|-------|------------|
| `meeting-action-tracker` | קורא סיכום ישיבה ומוציא רשימת action items — מי אחראי, מה המשימה, מה הדדליין |
| `churn-risk-analyzer` | סורק פניות תמיכה ומזהה לקוחות עם סימני עזיבה (churn signals) |
| `competitive-monitor` | קורא release notes של מתחרים ומסכם מה הם שחררו לאחרונה |

## חלק 2: יצירת Skill חדש

### מה זה Skill?

Skill הוא slash command שאתם מפעילים ידנית עם `/שם`. הוא רץ בתוך השיחה הראשית (לא בהקשר מבודד). Skill מתאים במיוחד ל:

- פורמט פלט קבוע שאתם רוצים שייראה אותו דבר כל פעם
- משימות חוזרות שאתם שולטים בעיתוי שלהן
- תהליכים שרוצים להפעיל בלחיצה אחת

### הפרומפט — הדביקו אותו ישירות ל-Claude Code:

```
אני רוצה ליצור skill חדש ב-Claude Code (slash command).

שם: [תאר שם קצר באנגלית, לדוגמה: /sprint-retro]
מטרה: [מה ה-skill אמור לעשות כשמפעילים אותו]
קלט: [מה יהיה למשתמש כשמפעיל? טקסט? קובץ?]
פלט: [מה הפורמט המדויק שאתה רוצה?]
שאלות הבהרה: [האם ה-skill צריך לשאול שאלות לפני שמתחיל? כן/לא]

תיצור עבורי skill שכולל:
- כותרת ומטרה ברורה
- הוראות שלב אחר שלב
- תבנית פלט מדויקת (בעברית)
- דוגמה לשימוש

שמור את הקובץ ב-.claude/skills/[שם].md
```

### רעיונות ל-Skills למנהלי מוצר

| Skill | מה הוא עושה |
|-------|------------|
| `/sprint-retro` | הופך הערות retrospective לא-מסודרות לתוכנית action מובנית |
| `/feature-brief` | לוקח bullet points ויוצר דף feature brief מסודר לסטייקהולדרים |
| `/release-notes` | הופך רשימת פיצ'רים שנשלחו ל-release notes שמדברים בשפת המשתמש |

## חלק 3: טיפים לפני שמתחילים

**שדה ה-description הוא הכי חשוב ב-Agent.**
Claude משתמש בו כדי להחליט מתי להפעיל את ה-agent. כתבו אותו בפורמט: "Use this agent when..." — ככל שתהיו ספציפיים יותר, ה-agent יופעל בזמן הנכון.

**קבצי Skill — שמרו אותם קצרים.**
עד 500 שורות לקובץ. אם יש חומר עזר (templates, דוגמאות ארוכות), שימו אותו בקובץ נפרד והפנו אליו מה-skill.

**תבדקו לפני שתסמכו.**
לפני שמשתמשים ב-agent או skill לעבודה אמיתית, תבדקו אותו עם בקשה פשוטה. ראו שהפלט נראה כמו שציפיתם.

**שמות באנגלית, הוראות בעברית — שתי השפות יחד עובדות מצוין.**
קראו ל-agent שלכם `churn-detector`, אבל כתבו את כל ההוראות הפנימיות בעברית. Claude מבין שניהם בלי בעיה.

**agent אחד = משימה אחת. Skill אחד = פורמט אחד.**
אל תנסו לדחוף הכל לתוך כלי אחד. עדיף שלושה agents פשוטים שעובדים, מאשר agent אחד "אוניברסלי" שמתבלבל.
