תיאור: קריאה ל-GET /api/v2/tasks?page=2 מחזירה את אותן 25 משימות כמו page=1. הפרמטר page מתקבל מהבקשה אך לא מועבר לשאילתת ה-database — הקוד תמיד מחזיר LIMIT 25 OFFSET 0 ללא תלות בפרמטר. הבעיה קיימת מיום ההשקה של API v2.
השפעה: כל לקוחות API שמנסים לאחזר יותר מ-25 משימות — 8 שותפי אינטגרציה, ~3,000 משתמשי קצה.
