# מבנה הקבצים של Claude Code — מדריך עזר לפרודקט מנג'רים

---

## סעיף 1: שני "בתים" — פרויקט מול אישי

ל-Claude Code יש שני מקומות שבהם הוא שומר הגדרות ומידע:

**`.claude/` בתוך הפרויקט** — משותף עם הצוות, עולה ל-Git.
**`~/.claude/` על המחשב שלך** — רק שלך, חל על כל הפרויקטים.

**אנלוגיה:** תחשוב על זה כמו Google Drive משותף של הצוות לעומת הדסקטופ האישי שלך.
- מה שנמצא ב-Google Drive המשותף — כולם רואים.
- מה שנמצא אצלך על הדסקטופ — רק אתה.

## סעיף 2: הקובץ הכי חשוב — CLAUDE.md

### מה זה CLAUDE.md?

`CLAUDE.md` הוא מסמך ה-onboarding שאתה כותב עבור Claude.

כשאתה פותח סשן ב-Claude Code, הוא קורא את הקובץ הזה אוטומטית — בלי שתבקש. זה כמו תדריך שאתה נותן לקבלן חדש לפני שהוא מתחיל לעבוד: "הנה מי אנחנו, הנה איך אנחנו עובדים, הנה מה חשוב לזכור."

### איפה הוא יושב?

- **הנפוץ ביותר:** בשורש הפרויקט — `./CLAUDE.md`
- **אפשרות נוספת:** בתוך `.claude/CLAUDE.md`

### CLAUDE.local.md — הגרסה האישית שלך

לפעמים יש לך הערות שאתה רוצה להוסיף ל-CLAUDE.md של הפרויקט, אבל לא לשתף עם הצוות — למשל, הדרפים של הדמו שלך, או הגדרות אישיות.

`CLAUDE.local.md` = הוספות שלך לפרויקט שלא עולות ל-Git. הוא נכנס אוטומטית ל-gitignore.

### חשוב מאוד: CLAUDE.md הוא לא memory

| | CLAUDE.md | Memory |
|--|-----------|--------|
| מי כותב? | **אתה** | **Claude** |
| מתי נכתב? | לפני שמתחילים | תוך כדי עבודה |
| מה יש בו? | הקשר, כללים, הסברים | פקודות שעבדו, העדפות שלמד |

CLAUDE.md = אתה כותב עבור Claude. Memory = Claude כותב עבור עצמו. (על זה בסעיף 6.)

## סעיף 3: Agents — עוזרים אוטונומיים

### מה זה Agent?

Agent הוא מומחה אוטונומי שעובד בנפרד — עם הקשר משלו, מבודד מהשיחה הראשית.

**אנלוגיה:** תחשוב על Agent כמו קבלן שאתה שולח לעשות עבודה ספציפית. אתה מסביר לו מה צריך, הוא הולך לעשות את זה, ומחזיר לך את התוצאות. הוא לא יושב בישיבה איתך — הוא עובד בנפרד.

### איך Claude יודע מתי להשתמש ב-Agent?

Claude מחליט לבד, על פי תיאור המשימה שנתת. אם המשימה נראית כזו שמתאים לה Agent — הוא ישתמש בו.

### איפה Agents יושבים?

- **ברמת הפרויקט:** `.claude/agents/` (משותף עם הצוות)
- **ברמה האישית:** `~/.claude/agents/` (רק שלך, לכל הפרויקטים)

### איך נראה קובץ Agent?

קובץ `.md` עם כותרת YAML בראש:

```markdown
---
name: bug-analyzer
description: מנתח דוחות באגים ומסווג לפי דחיפות
tools: read, search
model: claude-opus-4-5
---

אתה מנתח באגים מנוסה. קרא את הדוח, סווג כל באג לפי דחיפות...
```

### מגבלה חשובה

Agents לא יכולים ליצור Agents אחרים. כל Agent עובד לבד.

## סעיף 4: Skills — פקודות שאתה מפעיל

### מה זה Skill?

Skill הוא פקודה שאתה מפעיל עם `/שם-הפקודה`.

בניגוד ל-Agent שעובד בנפרד — Skill רץ בתוך השיחה הראשית שלך, רואה את כל ההיסטוריה.

**דוגמה:** `/write-stakeholder-update` — פקודה שמייצרת עדכון לבעלי עניין. אתה מפעיל אותה, Claude מבצע אותה ומחזיר תוצאה — הכול בתוך אותה שיחה.

**הערה:** Skills ו-slash commands הם אותו דבר בדיוק. מינואר 2026 המושגים מאוחדים.

### איך להפעיל?

- `/שם-הסקיל` — אתה מפעיל ידנית
- Claude יכול גם להפעיל אוטומטית כשהוא מזהה שזה רלוונטי

### איפה Skills יושבים?

- **ברמת הפרויקט:** `.claude/skills/` (משותף עם הצוות)
- **ברמה האישית:** `~/.claude/skills/` (רק שלך, לכל הפרויקטים)

### איך נראה קובץ Skill?

קובץ `.md` פשוט עם הוראות:

```markdown
# Write Stakeholder Update

כשמישהו מבקש עדכון לבעלי עניין, כתוב:
1. סיכום של מה בוצע
2. מה עומד לקרות
3. אם יש בעיות — ציין אותן בפירוש
```

## סעיף 5: Agents מול Skills — מתי להשתמש בכל אחד?

| | Agents | Skills |
|--|--------|--------|
| רץ ב... | הקשר מבודד | השיחה הראשית |
| מי מפעיל? | Claude מחליט (או אתה מבקש) | אתה מקליד `/שם` |
| הכי טוב עבור | מחקר פתוח, עבודה אוטונומית | פורמט ידוע, משימות חוזרות |
| דוגמה | ניתוח 50 באגים | `/write-stakeholder-update` |
| רואה היסטוריית שיחה? | לא (מבודד) | כן |

**כלל אצבע:**
- יש לך משימה שתיקח זמן ודורשת חיפוש ומחקר? → Agent
- יש לך פורמט קבוע שאתה חוזר עליו שוב ושוב? → Skill

## סעיף 6: Memory — מה Claude זוכר מעצמו

### איך זה עובד?

Claude Code לוקח הערות לעצמו תוך כדי עבודה. הוא כותב אותן ל-Memory באופן אוטומטי, וטוען אותן בתחילת כל סשן (200 השורות הראשונות).

### איפה Memory נשמר?

```
~/.claude/projects/<שם-הפרויקט>/memory/MEMORY.md
```

### מה Claude כותב שם?

- פקודות שעבדו בפרויקט הזה
- דפוסים שהוא למד
- העדפות שהזכרת לו

### אפשר לכוון אותו?

כן. אתה יכול להגיד לו:
> "תזכור שאנחנו משתמשים ב-pnpm ולא ב-npm"

והוא יכתוב את זה ב-MEMORY.md שלו.

### שוב — ההבדל מ-CLAUDE.md

| | CLAUDE.md | MEMORY.md |
|--|-----------|-----------|
| מי כותב? | אתה | Claude |
| על מה? | הקשר ורקע כללי | מה Claude למד תוך כדי |
| איפה? | בפרויקט | על המחשב שלך בלבד |

## סעיף 7: Settings ו-Rules — בונוס

### settings.json

קובץ הרשאות — מה Claude מורשה לעשות ומה לא.
- `settings.json` — ברמת הפרויקט (משותף)
- `settings.local.json` — ברמה האישית (לא משותף)

### .claude/rules/

תיקייה עם הוראות מודולריות שנטענות אוטומטית לפי נושא. במקום CLAUDE.md ארוך אחד, אפשר לחלק לקבצים לפי תחום — כל קובץ נטען כשרלוונטי.

## סעיף 8: מפת מבנה — סיכום חזותי מהיר

```
המחשב שלך (~/.claude/)              הפרויקט שלך (.claude/)
──────────────────────────           ───────────────────────
CLAUDE.md                            CLAUDE.md  ← משותף עם הצוות
agents/                              CLAUDE.local.md  ← שלך בלבד
  my-agent.md                        agents/
skills/                                bug-analyzer.md
  my-skill.md                        skills/
settings.json                          write-update.md
projects/                            settings.json  ← משותף
  <project>/memory/                  settings.local.json  ← שלך בלבד
    MEMORY.md  ← Claude כותב את זה
```

### מקרא

| קבצים ב... | נשמרים ב-Git? | מי רואה? |
|--|--|--|
| `.claude/` בפרויקט | כן | כל הצוות |
| `~/.claude/` על המחשב | לא | רק אתה |
| קבצים עם `.local.` | לא | רק אתה |
